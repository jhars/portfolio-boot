nXu
GPS & SMS Social App

Description:
Coordinated user-user matches via topic-preferences & Facebook authentication. When two or more users successfully match, they are notified with a Twillio SMS message if they are within a certain distance of one another.
Build Time: 1-week
Technologies Used: HTML5 Geo-location, JavaScript, Facebook Graph API, Twillio SMS text-notifications
Framework (Backend): Ruby on Rails
Responsibilities & Lessons Learned:
a) How to structure calls to a database from dynamic HTML5 geo-location coordinates using AJAX
b) Ruby on Rails is a fast development framework, but not necessarily optimized for mobile solutions.
c) Wrote an algorythm which would take into account the GPS coordinates of users and determinse whether or not to send a text-message to other users (but only if they matched based on category already).

Problem Solving

When developing web-applications, I beleive in choosing the right tool for the job. I like to think about the "end-user" first -- what is the application user's experience like? -- and identifify available resources 2nd -- what data do we have access to or need to create access to? Finally, I like to ask how can I present the information to the user in th most intuitive way.

Apps
