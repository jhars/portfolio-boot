Custom Helmet Stickers





Customizable Team Helmet Stickers

Preview
GitHub

Description:
After exploring 3-5 different ways to accomplish my vision, I decided to use HTML5 and Sprint Animations to re-create elements of factorycustom.com (helmet manufacturer). I wanted to create a tool which allows users to customize design elements of real-world objects; giving the user enough freedom to create designs while limiting options/tools to create user-friendly interface.
Instead of setting up a green-screen and creating a precise photography process, I used images from factorycustom.com; which were perfectly formatted as well. The User-Interface displays available tools & colors in the top-left corner, a location which most internet-users are comfortable tinkering with(user-friendly location).
Build Time: 1-Week
Technologies Used: HTML5 Canvas; CSS; Javascript;
Responsibilities Lessons Learned: a) 360-- learning how it work using XY-Axis-Grid coordinates (similar to GPS) to track your mouse movement
b) Mouse (click) Events on a laptop differ from Touch Events on mobile devces in the code (HTML5)
c) Using images to create custom UX/UI involves careful coordination of images, HTML, CSS and JavaScript

nXu -- GPS & SMS Social App

GPS & SMS text-message systems coordinated user-user matches via topic-preferences & Facebook authentication. When two or more users successfully match, they are notified with a Twillio SMS message if they are within a certain distance of one another.

Specs-- Build Time: 1-week ; Team Size: 4
Technologies Used: HTML5 Geo-location, JavaScript, Facebook Graph API, Twillio SMS text-notifications
Framework (Backend): Ruby on Rails
Responsibilities & Lessons Learned: a) How to structure calls to a database from dynamic HTML5 geo-location coordinates using AJAX
b) Ruby on Rails is a fast development framework, but not necessarily optimized for mobile solutions.
c) Wrote an algorythm which would take into account the GPS coordinates of users and determinse whether or not to send a text-message to other users (but only if they matched based on category already).
