WHAT I DO



Creative
Problem Solving
Problem Solving

When developing web-applications, I beleive in choosing the right tool for the job. I like to think about the "end-user" first -- what is the application user's experience like? -- and identifify available resources 2nd -- what data do we have access to or need to create access to? Finally, I like to ask how can I present the information to the user in th most intuitive way.



Mobile & Web
Apps
Web & Mobile Applicatons

I am a software-engineer and code in a range of 6 modern programming languages: JavaScript, Swift, Ruby, Python, HTML5, CSS3. I believe that the trend toward a more mobile web will continue which is why I have recently developed my accumen for building web applications. I develop software-applications in three different "stacks": Swift (iOS), MEAN & Ruby on Rails.



API-Driven
Architecture
API Data -> Device Proliferation

API-First Design is an emerging standard in today's data-interaction web. It serves as a live source of central data ("source of truth") which allows for real-time data updates acrosse multiple devices/platforms ("device proliferation"). I am versed in API and data-flow architecture with databases (MongoDB , PostgreSQL, CoreData-iOS).
